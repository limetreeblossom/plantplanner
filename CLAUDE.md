# Project Instructions

## Before committing to Git

Before committing, assess whether the change affects the UI (HTML, CSS, JS rendering, component logic).

- **UI change**: Ask the user if they want to verify it themselves in the browser, or if they want you to verify it. Wait for their answer before committing.
- **Non-UI change** (data scripts, JSON files, types, tests, config): Skip browser verification and commit directly.
